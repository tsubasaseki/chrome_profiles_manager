品質向上レポート

この文書は、品質向上作業で実施したログ追加、テスト追加、レビュー結果、修正内容をまとめたものです。

1. 実施概要

今回の品質向上作業では、次の対応を行いました。

  • 詳細なファイルログ出力を追加
  • 100件以上のテストケースを持つ自己完結テストランナーを追加
  • 潜在不具合のレビューと修正
  • 日本語プロファイル名の文字化け対策
  • 画面ログの DEBUG 抑制
  • 色分けとメモ用のメタ情報JSON対応
  • バックアップZIPへのメタ情報JSON同梱
  • バックアップZIPとログ出力エリアのリサイズ対応
  • 色選択ボタンの追加
  • 色付きプルダウン候補の追加
  • アイコン48px表示
  • 更新日時の n日前 表示
  • ZIP保存のバックグラウンド化と進捗表示
  • HTMLレポートの崩れにくい表レイアウト
  • docs の更新
  • Markdown と対応する SingleFileHTML の再生成

2. 追加したログ

ログは画面下部のログ欄に加えて、バックアップ先フォルダ配下にも保存します。

保存先は次の形式です。

  • <バックアップ先>\logs\ChromeProfilesManager_yyyyMMdd.log

追加した主なログは次の通りです。

  • アプリ起動情報
  • ログファイルの場所
  • プロファイル検出開始・完了
  • 検出した各プロファイルのディレクトリ、表示名、ユーザー、パス、サイズ
  • Local State 読み込み開始・完了・失敗
  • Preferences 読み込み開始・完了・失敗
  • アイコン画像読み込み開始・完了・失敗
  • サイズ計算開始・完了・警告
  • ZIP追加開始・完了・スキップ
  • ZIPバックアップ作成開始・完了
  • ZIP進捗の対象数、追加数、スキップ数
  • 隔離先フォルダ作成
  • 未処理エラー

ログレベルは INFO、DEBUG、WARN、ERROR を使います。

画面ログには INFO、WARN、ERROR を表示します。

DEBUG はファイルログにだけ保存します。これにより、アイコン画像読み込みなどの詳細ログで画面が埋まることを防ぎます。

5. レビューで見つけた潜在不具合

レビューとテストで、次の潜在不具合を見つけました。

5.1 1件だけのプロファイルが単体オブジェクト化する問題

PowerShell 関数は配列を返しても出力列挙するため、プロファイルが1件だけのとき、呼び出し側で単体オブジェクトとして扱われる可能性がありました。

修正として、配列を返す箇所で unary comma を使い、配列として保持するようにしました。

対象は次の通りです。

  • Get-ChromeProfiles
  • Get-SelectedProfilesFromGrid
  • Get-AllProfilesFromGrid

5.2 Runspace の EndInvoke 結果が配列1個になる問題

Runspace 非同期読み込みで、EndInvoke() の結果が配列1個を含む配列として返る可能性がありました。

修正として、UI一覧へ渡す前に結果を正規化する処理を追加しました。

5.3 隔離先フォルダの同名衝突

同じ秒に隔離処理を複数回実行した場合、隔離先フォルダが同名衝突する可能性がありました。

修正として、同名フォルダが存在する場合は yyyyMMdd_HHmmss_01 のように連番を付けて回避するようにしました。

5.4 バックグラウンド処理のログ不足

Runspace 側の非同期読み込みで、詳細なログが残りにくい状態でした。

修正として、Runspace 側にも Write-UiLog とログファイルパスを渡し、UIには触れずファイルログへ記録できるようにしました。

5.5 日本語プロファイル名の文字化け

PowerShell 5.1 の Get-Content 既定エンコーディングに依存すると、Chrome の Local State や Preferences に含まれる日本語が文字化けする可能性がありました。

修正として、Read-Utf8JsonFile を追加し、Chrome の JSON を UTF-8 固定で読み込むようにしました。

5.6 画面ログの DEBUG 過多

詳細ログを追加した結果、アイコン画像読み込みなどの DEBUG ログが画面を埋める問題がありました。

修正として、画面ログには DEBUG を表示せず、ファイルログにだけ保存するようにしました。

5.7 プロファイル整理用のユーザーメモ不足

大量のプロファイルを整理する際、Chrome 由来の表示名だけでは判断しづらい問題がありました。

対応として、このアプリ専用の10色分類、メモ1、メモ2を追加しました。

保存先は <User Data>\_ChromeProfilesManager\profile_metadata.json です。

このJSONはバックアップZIPにも ChromeProfilesManager/profile_metadata.json として同梱します。

5.8 画面領域の固定

バックアップZIP欄とログ出力欄の高さが固定で、作業状況に合わせて調整しづらい問題がありました。

対応として、プロファイル一覧、バックアップZIP、ログ出力を SplitContainer で分割し、境界をドラッグしてサイズ変更できるようにしました。

5.9 色選択の分かりづらさ

色のドロップダウンだけでは、どこで色を変えるのか分かりづらい問題がありました。

対応として、ツールバーに 選択行の色 と10色ボタンを追加しました。選択中の行に対して、ボタンから直接色を反映できます。

さらに、色列のドロップダウン候補も色付きで表示し、候補ごとの背景色と色見本を見ながら選べるようにしました。

5.10 アイコンの視認性

24px相当のアイコンでは、プロファイルの識別に使うには小さい問題がありました。

対応として、アイコン画像を 48px x 48px にリサイズし、一覧の行高も広げました。

5.11 更新日時だけでは古さが分かりづらい

日時だけでは、どれくらい前のプロファイルか判断しづらい問題がありました。

対応として、更新日時を yyyy-MM-dd HH:mm:ss（n日前） の形式で表示するようにしました。

5.12 ZIP保存中にGUIが固まる問題

ZIPバックアップ作成をUIスレッドで同期実行していたため、大きなプロファイルではGUIが固まって見える問題がありました。

対応として、ZIPバックアップ作成を Runspace 上で実行し、UI側は WinForms Timer で進捗状態を読み取るようにしました。

ファイル数確認中は進捗バーを動き続ける表示にし、総ファイル数が分かった後は百分率、処理済み件数、追加数、スキップ数を表示します。

5.13 HTMLレポートの表が崩れる問題

HTMLレポートの列幅指定が合計100%を超えており、12列の表と長いパスが重なって大きく崩れる問題がありました。

対応として、表を横スクロール可能な .table-wrap で包み、colgroup と最小幅で列幅を安定させました。

未設定色のときに空の background: を出力しないようにし、長い文字列は折り返せるようにしました。

6. 残る注意点

現状の注意点は次の通りです。

6.1 Chrome 起動中バックアップ

Chrome 起動中にバックアップすると、ロック中のファイルがスキップされる可能性があります。

現在の解決策は、Chrome を終了してからバックアップすることです。

将来的な改善案として、VSS を使ったスナップショットバックアップがあります。

6.2 ZIP 作成のキャンセル

ZIP バックアップ作成はバックグラウンドで実行され、GUI全体は固まりにくくなっています。

ただし、現在の実装にはZIP作成のキャンセルボタンはありません。

将来的な改善案として、進行中のZIP作成をキャンセルするボタンを追加する方法があります。

6.3 完全削除機能

現在の実装は安全優先のため、完全削除ではなく隔離のみを行います。

完全削除が必要な場合は、ユーザーが隔離フォルダを確認してから削除します。

将来的な改善案として、二段階確認付きの完全削除機能があります。

7. 実行した検証

実行した検証は次の通りです。

  • tests/Run-Tests.ps1 による 265 件のテスト
  • Markdown と SingleFileHTML の対応確認
