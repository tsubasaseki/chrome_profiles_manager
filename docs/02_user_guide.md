# ChromeProfilesManager ユーザーガイド

この文書は、現在の実装に合わせた ChromeProfilesManager の使い方を説明します。

## 1. アプリの目的

ChromeProfilesManager は、Windows 上の Google Chrome プロファイルを整理するための GUI アプリです。

主な目的は次の通りです。

- 現在の Chrome プロファイルを確認する。
- プロファイル一覧の確認用 HTML を作成する。
- HTML を含む ZIP バックアップを作成する。
- 不要なプロファイルを完全削除ではなく隔離フォルダへ移動する。
- CCleaner などでキャッシュを削除した後、移行用の原本バックアップを作成する。

## 2. 起動方法

次のファイルをダブルクリックして起動します。

- `Start-ChromeProfilesManager.vbs`

この起動方法では、黒い console 画面を表示せずにアプリを起動します。

VBS ランチャーは、同じフォルダにある `ChromeProfilesManager.ps1` を PowerShell で起動します。

## 3. 既定の対象フォルダ

Chrome の User Data フォルダの既定値は次の通りです。

- `%LOCALAPPDATA%\Google\Chrome\User Data`

バックアップ先の既定値は次の通りです。

- `Desktop`

画面上部の `Chromeデータ` と `バックアップ先` の欄で、必要に応じて変更できます。

## 4. 画面の主な項目

画面上部には次の項目があります。

- `Chromeデータ`: Chrome User Data フォルダの場所
- `バックアップ先`: ZIP と HTML を保存するフォルダ
- `選択`: フォルダ選択ダイアログを開く
- `開く`: 指定フォルダを Explorer で開く

操作ボタンは次の通りです。

- `再読み込み`: 現在のプロファイル一覧を読み直す
- `Chrome終了`: Chrome に終了を依頼し、必要に応じて強制終了を確認する
- `HTML作成`: 現在の一覧から確認用 HTML を作成する
- `初回ZIP保存`: 初回バックアップ用の ZIP を作成する
- `清掃後ZIP保存`: CCleaner などで清掃した後の原本バックアップ ZIP を作成する
- `選択を隔離`: チェックしたプロファイルを隔離フォルダへ移動する
- `選択を開く`: 一覧で選択中のプロファイルフォルダを Explorer で開く
- `選択行の色`: 選択中のプロファイルへ、10色ボタンから色を直接設定する

ZIP保存中は、操作ボタン付近に進捗バーと進捗ラベルが表示されます。

ファイル数を確認している間は進捗バーが動き続ける表示になり、総ファイル数が分かると全体の進捗率、`処理済み件数 / 対象件数`、推定残り時間、追加数、スキップ数が表示されます。

進捗ラベルには個別ファイル名を表示しません。`ZIPへ追加中` の詳細なファイル名はログへ出力します。

ZIP 欄には次の操作があります。

- `更新`: バックアップ先フォルダ内の ZIP 一覧を更新する
- `内容確認`: 選択した ZIP の中身を一覧表示する
- `ZIPを開く`: 選択した ZIP ファイルを Explorer で開く

プロファイル一覧、バックアップZIP、ログ出力エリアの境界はドラッグで調整できます。

プロファイル一覧を広く見たい場合、またはログを広く見たい場合は、境界線を上下に動かしてください。

## 5. ログ

画面下部には、現在の操作ログが表示されます。

さらに、バックアップ先フォルダ配下の `logs` フォルダに日付別ログファイルを保存します。

ログファイル名の形式は次の通りです。

- `ChromeProfilesManager_yyyyMMdd.log`

ログには、次のような情報を記録します。

- アプリ起動情報
- 対象 User Data フォルダ
- バックアップ先フォルダ
- プロファイル検出開始・完了
- `Local State` と `Preferences` の読み込み結果
- サイズ計算開始・完了
- ZIP 追加開始・完了
- ロック中または読み取り不可でスキップしたファイル
- バックアップ作成開始・完了
- 隔離先フォルダ作成
- 未処理エラー

不具合が起きた場合は、このログファイルを確認すると原因を追いやすくなります。

画面下部のログ欄には、通常の操作確認に必要な `INFO`、`WARN`、`ERROR` を表示します。

`DEBUG` は画面には表示せず、ファイルログにだけ保存します。これにより、プロファイルのアイコン読み込みなどの詳細ログで画面が埋まることを防ぎます。

## 6. プロファイル一覧に表示される情報

プロファイル一覧には次の列があります。

- `アイコン`: ローカルに保存されているプロフィール画像。48px相当の大きさで表示します。
- `ディレクトリ`: `Default`、`Profile 1` などのフォルダ名
- `表示名`: `Local State` から取得した Chrome の表示名
- `ProfileName`: 各プロファイルの `Preferences` から取得した profile name
- `ログインユーザー`: 取得できる場合はメールアドレスなど
- `Google名`: 取得できる場合は Google アカウント名
- `アイコン情報`: Chrome 内蔵アバターなどの `avatar_icon` 識別子
- `色`: このアプリ専用の10色分類
- `メモ1`: このアプリ専用のメモ
- `メモ2`: このアプリ専用のメモ
- `サイズMB`: プロファイルフォルダの概算サイズ
- `更新日時`: プロファイルフォルダの最終更新日時。日時の後に `（n日前）` を表示します。
- `場所`: プロファイルフォルダのフルパス

Chrome 内蔵アバター画像そのものは Chrome の内部リソースであり、直接画像として表示できない場合があります。その場合も、`アイコン情報` には識別子を表示します。

Chrome の `Local State` と `Preferences` は UTF-8 として読み込みます。そのため、日本語の表示名や ProfileName も文字化けしにくくなっています。

## 7. 色分けとメモ

`色`、`メモ1`、`メモ2` は Chrome 本体の設定ではなく、ChromeProfilesManager 専用の情報です。

保存先は次の通りです。

- `<Chrome User Data>\_ChromeProfilesManager\profile_metadata.json`

色は次の10色から選べます。

- 赤
- 橙
- 黄
- 緑
- 水
- 青
- 紫
- 桃
- 灰
- 茶

`色`、`メモ1`、`メモ2` は一覧上で編集できます。

色は一覧の色付きドロップダウンから選べます。

色ドロップダウンの候補には、色名、Hex値、背景色、色見本が表示されます。

さらに、画面上部の `選択行の色` にある色ボタンを押すと、選択中のプロファイルへすぐ反映できます。

編集すると、`profile_metadata.json` に保存されます。

未設定に戻し、メモ1とメモ2も空にすると、そのプロファイルのメタ情報はJSONから削除されます。

## 8. 推奨作業手順

推奨する整理手順は次の通りです。

1. ChromeProfilesManager を起動する。
2. `再読み込み` でプロファイル一覧を確認する。
3. 必要に応じて `HTML作成` で確認用 HTML を作成する。
4. `初回ZIP保存` で最初のバックアップを作成する。
5. 不要なプロファイルにチェックを入れる。
6. `選択を隔離` で不要プロファイルを隔離フォルダへ移動する。
7. ユーザー自身で CCleaner などを使い、Chrome キャッシュを削除する。
8. `清掃後ZIP保存` で移行作業などに使う原本バックアップを作成する。

## 9. バックアップ ZIP の仕様

ZIP バックアップには次の内容が含まれます。

- `ChromeProfilesReport.html`
- `Local State`
- `ChromeProfilesManager/profile_metadata.json`
- `Profiles/<プロファイルディレクトリ名>/...` 配下の各プロファイルファイル

ZIP ファイル名にはステージと日時が含まれます。

例:

- `ChromeProfilesBackup_initial_20260630_001500.zip`
- `ChromeProfilesBackup_post-cleanup_20260630_002000.zip`

Chrome 起動中でもバックアップは実行できますが、ロック中のファイルがスキップされたり、不整合なバックアップになる可能性があります。そのため、Chrome 起動中は警告を表示します。

ZIP保存はバックグラウンドで実行されるため、保存中も画面のログ確認やウィンドウ操作ができます。

ZIP作成が完了すると、完成したZIPがバックアップZIP一覧で選択されます。完了ダイアログで確認すると、そのZIPの内容一覧をすぐ表示できます。

ZIP内に含まれる `ChromeProfilesReport.html` は、横スクロール可能な表として生成されます。列数が多く、メールアドレスやパスが長い場合も、ページ全体が大きく崩れにくい構造です。

ローカルに保存されているプロフィール画像がある場合、HTMLレポートには画像をData URIとして埋め込みます。そのため、HTML単体で画像付きのSingleFileHTMLとして閲覧できます。

## 10. 隔離の仕様

`選択を隔離` は、プロファイルを完全削除しません。

選択されたプロファイルは、Chrome User Data フォルダ配下の次の場所へ移動されます。

- `_ChromeProfilesManager_Quarantine\<日時>\<プロファイルディレクトリ名>`

Chrome 起動中の隔離操作はブロックされます。

バックアップ ZIP が存在しない状態での隔離操作もブロックされます。

同じ秒に隔離処理が複数回実行され、隔離先フォルダ名が衝突する場合は、末尾に連番を付けたフォルダへ退避します。

## 11. 非同期読み込み

起動直後のプロファイル読み込みは、GUI と分離して非同期で実行されます。

現在の実装では、PowerShell の Runspace を明示的に作り、WinForms の Timer で完了を監視します。

これにより、起動直後に画面が固まって見える問題を抑えています。

ZIP保存もバックグラウンドで実行されます。

これにより、大きなプロファイルを保存するときもGUI全体が固まりにくくなっています。

## 12. 注意事項

実プロファイルを扱う前に、必ず初回 ZIP バックアップを作成してください。

Chrome 起動中は、プロファイルファイルがロックされる場合があります。可能な限り Chrome を終了してからバックアップや隔離を行ってください。

隔離したプロファイルを完全に削除する機能は、現在の実装にはありません。不要であることを別途確認してから、ユーザー自身が隔離フォルダを削除してください。

バックアップ作成中は画面操作を一時的に無効化します。巨大なプロファイルでは時間がかかる場合があります。
